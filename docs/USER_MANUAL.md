## 1. 關於 BabyMakiSuk

BabyMakiSuk 是一款專為雙胞胎/多胞胎家庭設計的 Android 幼兒管理應用程式。提供以下核心能力：

- **雙孩同時管理**：同時追蹤兩個孩子的所有成長數據
- **成長曲線追蹤**：記錄身高、體重、頭圍，自動計算 WHO 百分位
- **就醫紀錄管理**：記錄就診資訊，AI 自動整理摘要與處方箋分析
- **疫苗提醒**：設定疫苗/就醫提醒，支援雙孩共用提醒
- **月報生成**：AI 自動彙整當月日誌、就醫、成長數據為月報
- **AI 智能助理**：兒科醫師、藥師、營養師等多角色 AI 諮詢
- **排便紀錄**：追蹤排便頻率，48 小時未排便自動提醒
- **Memo 記事**：附提醒功能的育兒備忘錄
- **深色模式**：支援系統跟隨、強制淺色/深色

## 2. 開始使用

### 2.1 首次啟動

首次啟動 App 時，系統會自動建立兩個預設孩子「小明」（男）與「小美」（女），並自動嘗試 Firebase 匿名登入（無需註冊即可使用）。

> ⚠️ **建議**：立即進入「設定」修改孩子姓名與生日，確保成長數據計算準確。

### 2.2 權限需求

| 權限 | 用途 | 必要 |
|------|------|------|
| 通知 (POST_NOTIFICATIONS) | 疫苗提醒、Memo 提醒、系統提醒 | 建議授予 |
| 儲存空間 | 處方箋圖片暫存 | 僅內部儲存，不需額外權限 |

### 2.3 底部導覽列

App 採用 5 個主要分頁：

```
醫護  |  就醫  |  首頁  |  成長  |  設定
```

- **醫護**：疫苗/就醫提醒管理
- **就醫**：就診紀錄與 AI 摘要
- **首頁**：儀表板總覽
- **成長**：身高體重生長曲線
- **設定**：App 設定、API 測試

### 2.4 側邊書庫

從任何頁面點擊左上角選單 ☰ 按鈕即可開啟側邊書庫，包含：

- 系統提醒（System Reminders）
- AI 精華卡（AI Insights）
- Memo 記事本
- 月報列表
- AI Portal 快捷入口

## 3. 首頁儀表板

### 3.1 孩子資訊卡

顯示兩個孩子的最新資訊：
- 姓名、月齡、生日
- 大頭貼（可點擊更換照片）
- 最新成長數據（體重、身高）

### 3.2 即時狀態區塊

- **排便紀錄**：顯示上次排便時間與距離現在時數
- **最新就診**：最近一次就醫摘要
- **今日 Memo**：當天的備忘錄列表
- **疫苗提醒**：下一劑預計施打時間
- **系統提醒**：未處理的系統提醒（如長時未排便警示）

### 3.3 快速操作按鈕

- 🚽 記錄排便：點擊即可記錄
- 📈 新增成長紀錄
- 🤖 AI 助理快速提問
- ✏️ 新增 Memo

### 3.4 排便提醒功能

系統會自動追蹤孩子排便時間：
- 距離上次排便超過 48 小時 → 產生「長時未排便」提醒
- 記錄排便後 → 自動標記所有相關提醒為已解決

## 4. 成長紀錄

### 4.1 查看成長曲線

1. 點擊底部「成長」分頁
2. 預設顯示男孩（若有雙胞胎）的成長曲線
3. 可切換孩子（點擊孩子選擇列）
4. 圖表顯示身高、體重、頭圍的歷史曲線

### 4.2 新增成長紀錄

1. 點擊右下角 + 按鈕
2. 填寫：日期、身高(cm)、體重(kg)、頭圍(cm，選填)
3. 可附加備註
4. 儲存後自動計算 WHO 百分位

### 4.3 AI 成長分析

1. 在成長紀錄列表頁，點擊 🤖 AI 分析按鈕
2. AI 會根據所有歷史數據，分析生長趨勢
3. 分析結果儲存為 AI 精華卡（可在書庫查看）

### 4.4 編輯/刪除

- 長按或點擊現有紀錄旁的編輯圖示進行修改
- 滑動或點擊刪除圖示移除紀錄

## 5. 就醫管理

### 5.1 記錄就診

1. 點擊底部「就醫」分頁
2. 點擊右上角 + 按鈕
3. 填寫：
   - 就診醫院、科別
   - 診斷結果
   - 備註（就診筆記）
   - 就診日期
   - 可拍照儲存處方箋
4. 儲存後資料同步至 Firestore 雲端

### 5.2 AI 就醫摘要

就醫紀錄儲存後，若勾選「圖片待 AI 分析」：

1. 系統後台會自動呼叫 AI 分析處方箋
2. 自動填入：診斷摘要、處方資訊、照護建議
3. 標記是否為緊急情況（isUrgent）

也可手動觸發 AI 摘要：
1. 在就醫列表中找到目標紀錄
2. 點擊「AI 摘要」按鈕
3. AI 自動分析就診備註並填入摘要欄位

### 5.3 處方箋圖片分析

1. 在新增/編輯就診時，拍照或選擇處方箋圖片
2. 填寫症狀提示（如：「咳嗽有痰三天」）
3. 點擊 AI 分析按鈕
4. AI 辨識圖片中藥名、劑量、用法
5. 分析結果進入「審閱」狀態，確認後自動填入對應欄位

### 5.4 緊急標記

AI 分析後若判斷為緊急（safetyFlag = "urgent"），系統會自動標記該筆就診紀錄，提示家長注意。

## 6. 健護提醒（疫苗）

### 6.1 新增提醒

1. 點擊底部「醫護」分頁
2. 點擊右上角 + 按鈕
3. 設定：
   - 提醒名稱（如：「B 肝第二劑」、「三合一疫苗」）
   - 預計日期
   - 提醒對象（可同時選兩個孩子）
   - 備註
   - 是否已完成
4. 儲存後顯示於列表

### 6.2 提醒狀態

- 🟢 綠色：已完成
- 🔴 紅色：已過期
- 🟠 橘色：7 天內即將到期
- 🔵 藍色：一般（尚有充足時間）

### 6.3 編輯與刪除

- 點擊編輯圖示修改提醒
- 點擊刪除圖示移除提醒
- 點擊 ✓ 圖示切換完成狀態

## 7. AI 智能助理

### 7.1 進入 AI Portal

可透過以下方式進入：
1. 側邊書庫 → AI Portal
2. 首頁快速按鈕 → AI 助理
3. 從成長分析頁面直接觸發

### 7.2 角色選擇

| 角色 | 適用情境 | 說明 |
|------|----------|------|
| 兒科醫師 | 健康諮詢 | 回答幼兒健康問題，建議就醫時機 |
| 藥師 | 用藥疑問 | 理解藥物成分、劑量與注意事項 |
| 營養師 | 副食品建議 | 依據月齡提供飲食規劃 |
| 發育分析師 | 成長數據分析 | 解讀百分位與發育趨勢 |
| 自由模式 | 一般問題 | 任何育兒相關問題 |

### 7.3 使用限制

- 每分鐘 5-10 次請求（依任務類型而定）
- AI 結果僅供參考，不構成醫療診斷建議
- 處方分析每分鐘上限 3 次

### 7.4 安全提示

> ⚠️ AI 結果僅供參考，不構成醫療診斷建議，請諮詢專業兒科醫師。

所有 AI 結果下方都會顯示此免責聲明。

## 9. 月報功能

### 9.1 自動提醒

每月最後 7 天，系統會自動產生「月底月報預報」提醒，提示家長填寫日誌。

### 9.2 月報生成

月報由 AI 自動彙整以下資料：
- 當月每日日誌（睡眠、飲食、便便、心情）
- 當月就診紀錄
- 當月系統提醒
- 成長快照（如該月有紀錄）

### 9.3 月報內容

- 📝 整月總結（200 字內）
- ⭐ 本月亮點（3 條）
- 💡 給家長的建議（2 條）
- 🔑 搜尋關鍵字（3-5 個，供 FTS 搜尋）

### 9.4 查看月報

1. 側邊書庫 → 月報列表
2. 或在「首頁」→ 每月報告
3. 可按年份、關鍵字搜尋

## 11. 常見問題 (FAQ)

### Q1: 資料會同步到雲端嗎？

A: 就醫紀錄會同步上傳至 Firebase Firestore。成長紀錄、Memo、日誌等目前僅儲存於本機 Room 資料庫。

### Q2: 如何更換孩子的大頭貼？

A: 在「設定」→ 點擊孩子姓名 → 點擊大頭貼區域 → 選擇照片。

### Q3: AI 分析結果不準確怎麼辦？

A: AI 結果僅供參考。您可手動編輯 AI 填入的診斷摘要、處方與照護建議。

### Q4: 通知沒有彈出？

A: 確認已在系統設定中允許 BabyMakiSuk 的通知權限（Android 13+ 需手動授予）。

### Q5: 如何清除所有資料？

A: 進入 Android 設定 → 應用程式 → BabyMakiSuk → 儲存空間 → 清除資料。注意此操作無法復原。

### Q6: 支援多少個孩子？

A: 目前以雙胞胎為設計核心，理論上可支援 2+ 個孩子，但 UI 雙欄佈局最適合兩個孩子。

## 12. 附錄 A：技術架構總覽

### 12.1 技術棧

| 層級 | 技術 |
|------|------|
| 語言 | Kotlin |
| UI | Jetpack Compose + Material 3 |
| DI | Hilt |
| 資料庫 | Room (SQLite) |
| AI | Google Gemini API / Gemma |
| 雲端 | Firebase Auth / Firestore / Storage |
| 背景 | WorkManager |

### 12.2 模組結構

```
app/              → MainActivity、Application、Navigation、DI
core/model/       → 純 Kotlin Domain Model（無 Android 依賴）
core/data/        → Room Entity / DAO / Repository / Migration / Worker
core/ui/          → 共用 Composable、Theme
core/ai/          → AiDispatcher、Prompt Builder、RateLimiter
core/firebase/    → Firebase Auth / Firestore / Storage
core/drive/       → Google Drive（保留，未實作）
feature/ai/       → AI Portal Screen
feature/home/     → 首頁儀表板
feature/growth/   → 成長紀錄 + 曲線
feature/medical/  → 就醫管理 + 發燒紀錄
feature/library/  → 書庫（Memo、精華卡、提醒、月報）
feature/vaccine/  → 疫苗提醒
feature/weeklyreport/ → 月報
feature/settings/ → 設定
```

### 12.3 MVI 架構

每個 Feature 遵循 MVI 模式：

```
User Action → ViewModel (Intent) → Repository → ViewModel (State) → UI
                  ↓
            UiEvent (Toast, Navigation)
```

- **UiState**：`sealed interface` 定義 Loading / Success / Error
- **UiEvent**：`SharedFlow` 傳遞一次性事件

### 12.4 AI Fallback Chain

AI 請求透過 AiDispatcher 按以下優先順序嘗試模型：

```
MEDICAL_CONSULTATION:
  Gemma 4 31B → Gemini 3.1 Flash Lite → Gemma 4 26B

MEDICAL_OCR:
  Gemini 3 Flash → Gemini 3.1 Flash Lite → Gemma 4 31B

QUICK_CHAT / CUSTOM_PRESET:
  Gemini 3.1 Flash Lite → Gemma 4 31B → Gemma 4 26B
```

### 12.5 資料留存策略

| 資料類型 | 保留期限 |
|----------|----------|
| 日誌 (DailyLog) | 90 天（受月報保護） |
| 排便紀錄 (ToiletRecord) | 60 天（受月報保護） |
| AI 精華卡 | 180 天 |
| 系統提醒 | 觸發後自動清除 |
| 疫苗提醒 | 完成後自動清除 |
| Firebase Storage 圖片 | 1 年後清除 |
| 處方箋暫存圖片 | 7 天後清除 |

## 13. 附錄 B：資料安全與隱私

### 13.1 本機儲存

- 所有幼兒資料以 SQLite（Room）儲存在裝置內部儲存空間
- 不強制網路連線，所有核心功能可離線使用
- API Key 透過 BuildConfig 注入，編譯後難以逆向取出

### 13.2 雲端傳輸

- 與 Gemini API 的通訊使用 HTTPS 加密
- 就醫資料同步至 Firebase Firestore 使用傳輸層加密
- Firebase Storage 圖片存取需經過身份驗證

### 13.3 AI 資料處理

- 傳送至 AI 的資料包含幼兒的月齡、性別、過敏史
- **不傳送**可識別個人的姓名、住址等資訊
- AI 分析結果僅供參考，非醫療診斷依據
- 所有 AI 請求受 RateLimiter 控制，防止 API 濫用

### 13.4 資料刪除

- 使用者可隨時在本機刪除任何紀錄
- 清除 App 資料即可刪除所有本機資料
- Firebase 雲端資料需另行請求刪除
